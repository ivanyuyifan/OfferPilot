      OfferPilot 使用帮助

      从配置到复盘的完整指南
      OfferPilot 不替你生成“完美答案”，而是帮助你识别真实经历中的证据缺口、岗位匹配问题和表达改进空间。

      🚀 快速开始

      1. 在右上角配置 API Endpoint、API Key 和模型。
      2. 上传 .txt、.md、.docx 或文本型 PDF 简历，也可以直接粘贴文本。
      3. 粘贴目标岗位 JD，生成职责、能力地图与 5 道训练问题。
      4. 逐题回答，完成追问与四维诊断。
      5. 修改回答、重新评分、导出或保存记录。

      📄 文件上传

      • Word 请使用 .docx；旧版 .doc 需先另存为 .docx。
      • PDF 必须包含可复制文字，扫描版需先进行 OCR。
      • 单个文件最大 20 MB。
      • 文件只在当前浏览器中解析，不会单独上传到其他服务器。

      ⚙ API 配置

      • Endpoint 需填写完整的聊天补全地址。
      • 接口必须允许浏览器跨域访问。
      • 单次请求最长等待 90 秒。
      • 配置仅保存在当前浏览器，请勿在公共设备保存密钥。

      🎙 面试规则

      • 优先训练简历证据较弱的能力。
      • 缺少场景、行动、数据或结果时会被追问。
      • 每题最多追问 2 次，之后必须进入诊断。
      • Enter 发送，Shift + Enter 换行。

      📊 评分说明

      • 岗位匹配度：是否回应岗位核心要求。
      • 证据完整度：是否有场景、行动与结果。
      • 表达清晰度：结构和重点是否清楚。
      • 可信度：细节是否具体、一致且可追问。
      • 所有评分均为 0–100 分制。

      🗂 记录与导出

      • 诊断后需点击“保存到训练记录”。
      • 最多保留最近 20 次训练。
      • 80 分以上回答可标记为参考答案。
      • 清除浏览器数据会同时清除本地记录。

      💡 回答建议

      • 先交代场景与目标，再说明个人行动。
      • 尽量提供时间、规模、比例或效率变化。
      • 区分团队成果和自己的具体贡献。
      • 最后补充结果、复盘和岗位关联。

      🛠 常见问题

      • 网络请求失败：检查网络、接口地址和跨域设置。
      • 请求超时：请求超过 90 秒，请稍后重试。
      • AI 返回格式异常：模型未返回要求的 JSON，请重试。
      • 扫描版 PDF 无文字：先使用 OCR 转成可搜索 PDF，或直接粘贴简历文本。
      • 评分异常低：新版会校验百分制；非 0–100 量纲会明确报错。
      • 图表或 PDF 失败：检查网络后刷新，确认 CDN 依赖已加载。
